1.查看当前项目的所有分支
git branch -a
* 是当前所在分支
master是主分支
remotes/origin/xxx的是远程分支

2.分支的一般架构
master是主分支,是发布重大版本的分支,需要远程分支
develop是开发分支,日常开发应该在这个分支上,完成后并入master分支,需要远程分支
feature是功能分支,从develop分支上分出来,完成后并入develop分支,命名可以采用feature-*的形式
release是预发布分支,预发布结束后必须合并到develop和master分支,命名可以采用release-*的形式

3.切换并创建新分支
git checkout -b 分支名

4.切换支付
git checkout 分支名

5.提交分支数据到远程分支
git commit "本次提交的信息"
git push origin 本地分支:远程分支(可以是新分支)
或者git push origin 远程分支(可以是新分支) 默认是当前分支

6.分支合并
假设要开发一个国庆的活动页,先创建一个actitity-guoqin的分支
git checkout -b activity-guoqin
开发完activity-guoqin.js后,切换到develop-1分支
git checkout develop-1
把activity-guoqin分支合并到develop-1分支
git merge activity-guoqin
提交develop-1分支到远程develop-1分支
git add -A (添加所有修改)
git status (查看添加的内容)
git commit -m "本次提交"
git push origin develop-1
自测完毕并且提交所有develop-1分支的代码后合并到master分支
git checkout master (可能需要重新打开git base)
git merge develop-1
git add -A (添加所有修改)
git status (查看添加的内容)
git commit -m "本次提交"
git push origin master
