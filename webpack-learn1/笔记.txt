1.npm 安装依赖包通常是'^','^2.0.0'可能真实版本是'2.5.13'
npm查看依赖包的真实版本: npm list [packagename],如npm list webpack

2.如果在构建过程中出现Cannot find module 'xxx',而'xxx'已经安装了的,说明某一个包依赖'xxx'的版本
要高于'xxx'包的当前版本,可以升级'xxx'包,也可以找到并降低那个依赖'xxx'的包的版本,一般做法处理插件包

3.npm info webpack@3.10.0可以查看webpack@3.10.0支持的依赖包(有名的),
npm info webpack-dev-server@3.1.0(插件包) 可以看到依赖的webpack是'^4.0.0',所以不适用
构建的时候会报错,这时候需要把webpack-dev-server卸载重新下载低版本的
然后npm info webpack-dev-server@2.3.0(存在这个版本才会打印信息),发现适用当前webpack版本
然后npm install webpack-dev-server@2.3.0 删除3.1.0版本的并且下载2.3.0版本的
